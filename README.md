# 💊 Трекер приёма лекарств и контроля симптомов (Medication & Symptom Tracker)

Минимально жизнеспособный продукт (MVP) клиентского веб-приложения для контроля расписания приёма медикаментов, ведения дневника симптомов и управления профилями пациентов.

## ✨ Ключевые возможности

*   👥 **Управление профилями (Multi-profile):** Поддержка нескольких пациентов в одном приложении. Плавное переключение, добавление и безопасное удаление профилей.
*   💊 **Диспетчеризация назначений:** Динамическое добавление новых препаратов. Гибкая настройка: название, точная дозировка (мг, мл, капсулы), время приема и специфика пищевой совместимости.
*   ⏰ **Умный трекинг времени:** Автоматический расчет пропущенных приёмов. Визуальная индикация: карточка лекарства меняет цвет, если время приёма прошло, а оно еще не принято.
*   📝 **Дневник симптомов и приёмов:** Ежедневная фиксация самочувствия для отслеживания динамики здоровья и точное логирование факта приёма лекарств.
*   ⚡ **Адаптивный UI и безопасность:** Использование `Tailwind CSS` для идеального отображения на любых устройствах. Защита от потери данных (подтверждения действий).

## 🛠 Стек технологий

*   **Frontend Library:** React 18 (Functional Components, Hooks)
*   **Стилизация:** Tailwind CSS
*   **Типизация:** TypeScript
*   **Сборщик:** Vite

## 🚀 Запуск проекта локально (Getting Started)

Для запуска проекта на вашем локальном компьютере, выполните следующие шаги:

1. Убедитесь, что у вас установлен [Node.js](https://nodejs.org/).
2. Склонируйте конфигурацию репозитория на ваш компьютер:
   ```bash
   git clone <ССЫЛКА_НА_ВАШ_GITHUB_РЕПОЗИТОРИЙ>
   ```
3. Перейдите в скачанную директорию:
   ```bash
   cd <ИМЯ_ПАПКИ_ПРОЕКТА>
   ```
4. Установите необходимые зависимости:
   ```bash
   npm install
   ```
5. Запустите локальный сервер для разработки:
   ```bash
   npm run dev
   ```
6. Откройте в браузере предоставленную в консоли ссылку (обычно `http://localhost:3000` или `http://localhost:5173`).

## 🏗 Архитектура и хранение данных (Mock Data)

На текущем этапе развития (Frontend MVP) приложение сфокусировано на пользовательском интерфейсе и UX-взаимодействии. В качестве источника данных выступает **In-Memory Mock Database** — локальная база, инициализируемая в оперативной памяти браузера.

Такой подход позволяет демонстрировать и тестировать все CRUD-операции без сетевых задержек. Архитектура приложения (использование изоляции React-состояний) изначально спроектирована таким образом, чтобы в будущем этот локальный слой данных мог быть быстро и гладко заменен на интеграцию с полноценным Backend REST API сервером и настоящей базой данных (например, PostgreSQL).
